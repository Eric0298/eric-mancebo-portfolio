## Architecture principles

The code must be:

- Modular
- Clean
- Scalable
- Typed
- Component-based
- Easy to inspect with DevTools
- Easy to maintain
- Prepared for future automation and chatbot features

Do not place everything in one file.

Separate:

- layouts
- pages
- sections
- UI components
- cards
- forms
- effects
- mobile components
- data
- hooks
- lib
- styles
- types
- web-components

## Visual direction

The final art direction is not defined yet.

Do not create a definitive visual identity.

Use a neutral, flexible and technical base.

Use:

- CSS variables
- design tokens
- reusable classes
- responsive utilities
- scalable spacing system

Avoid:

- hardcoded random colors
- chaotic gradients
- generic template look
- excessive animations
- visual decisions that are difficult to change later

The final visual direction will be decided later using reference screenshots.

## Responsive and motion strategy

The website must not be only responsive in a basic way.

It must have a carefully planned desktop and mobile experience.

### Desktop

Desktop should support a premium visual experience:

- Layered parallax
- Scroll reveal
- Floating cards
- Depth effects
- Dashboard/mockup sections
- Smooth transitions
- Editorial SaaS-like composition
- Optional mouse-based microinteractions

### Tablet

Tablet should use:

- Reduced parallax
- Simplified motion
- Balanced spacing
- Clear layout density

### Mobile

Mobile must be intentionally designed for people who open the portfolio quickly from LinkedIn or a CV.

Mobile priorities:

- Fast loading
- Clear hierarchy
- Thumb-friendly navigation
- Sticky contact actions
- Simplified motion
- No heavy parallax
- Reduced animations
- Excellent readability
- Shorter sections
- Clear calls to action

Responsive does not mean shrinking the desktop website.

Responsive means redesigning the experience according to the device.

## Accessibility and motion

Motion must never harm:

- Performance
- Accessibility
- Readability
- Usability

Respect:

- prefers-reduced-motion
- keyboard navigation
- semantic HTML
- contrast
- readable font sizes
- accessible buttons and links
